# Delivery Script

## Opening: 0–3 Minutes

Today I want to share a very practical idea.

Many Informatics students are excited to develop applications. That is good. However, one common weakness is that students often jump too quickly into coding. They want to show that they can build a login page, dashboard, database, or AI function. But sometimes, they cannot explain clearly what problem they are solving.

So today, we will focus on how to move from a scenario to a clear application idea using generative AI.

## Main Explanation: 3–15 Minutes

When you receive a project idea, do not ask first:

> Should I use Python, PHP, Laravel, React, or Flutter?

The first question should be:

> What is happening in the real world?

For example, if you say:

> I want to build an attendance system.

That is still too general.

You need to explain the scenario.

Is the attendance process slow?  
Is there cheating?  
Are records missing?  
Is the lecturer spending too much time preparing reports?  
Is the problem faced by students, lecturers, or admin staff?

Once the scenario is clear, the application becomes easier to design.

A good scenario should mention:

1. The user
2. The context
3. The current process
4. The pain point
5. The data
6. The impact
7. The proposed application support

## Generative AI Workflow: 15–23 Minutes

Generative AI can help you refine your idea. But the quality of the answer depends on the quality of the scenario.

Use ChatGPT or Claude to analyse the problem. Ask it to identify the users, pain points, requirements, and possible database structure.

Then use Google Stitch to generate UI design ideas. This is useful because students can see how the system might look before coding.

After that, use Google Sheets as a simple database if the project is still at prototype level.

Finally, use GitHub to document your work properly.

The important thing is not to use AI blindly. AI should help you think, not replace your thinking.

## Mini Activity: 23–28 Minutes

Now let us try one simple example.

Weak idea:

> I want to build a food ordering app for campus.

This is not enough. We need to improve the scenario.

Ask:

1. Who has the problem?
2. What happens now?
3. What is inefficient?
4. What data is involved?
5. What should the app improve?

Then the idea becomes clearer:

> Students face long queues during lunch hours. Some do not know which stalls are open or which menu items are available. Stall operators handle orders manually, which causes delays. A campus food pre-ordering app can help students view menus, place orders, and check order status.

Now we have a real project direction.

## Closing: 28–30 Minutes

The key lesson is simple.

Do not start with technology. Start with the problem.

A strong student can explain the scenario clearly before building the application. Generative AI can help you clarify the problem, generate requirements, design screens, and prepare a simple prototype. But the student must still understand the real situation.

Before you build, ask:

> What problem am I really solving?
