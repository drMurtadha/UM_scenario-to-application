# UM_scenario-to-application

# From Scenario to Application: Problem Formulation Using Generative AI

## Training Overview

This repository contains teaching material for a 30-minute training session for **S1 students of Teknik Informatika, Fakultas Teknik, Universitas Negeri Malang**.

The session introduces a practical method for helping students move from a raw scenario to a clearer application development idea using generative AI tools such as **ChatGPT**, **Claude**, **Google Stitch**, **GitHub**, and **Google Sheets**.

## Main Theme

Many Computer Science, Informatics, IT, and Software Engineering students are eager to demonstrate their technical ability by building applications. However, a common weakness is that they start from the technology or interface before understanding the real problem.

This session argues that students should begin from the **scenario**. The scenario helps them identify the real users, current process, pain points, required data, expected system functions, and realistic application scope.

## Learning Outcomes

By the end of the session, students should be able to:

1. Explain why problem understanding must come before application development.
2. Convert a vague application idea into a clearer problem scenario.
3. Identify users, pain points, data needs, and system requirements from a scenario.
4. Use generative AI to refine problem statements and application ideas.
5. Prepare a simple prototype plan using Google Sheets, Google Stitch, and GitHub.

30 minutes.

## Target Audience

## Core Message

> A good application is not built from coding skills alone. It is built from a clear understanding of the problem.

## Recommended Tools

| Tool | Purpose |
|---|---|
| ChatGPT | Scenario clarification, requirements, project title, user stories |
| Claude | Structured project reasoning, document drafting, requirement refinement |
| Google Stitch | UI mockup generation from scenario prompts |
| Google Sheets | Simple prototype database |
| GitHub | Documentation, version control, code repository, project evidence |
