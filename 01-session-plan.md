# 30-Minute Session Plan

## Session Title

**From Scenario to Application: A Quick Way to Formulate Problems Using Generative AI**

## Target Audience

S1 students of Teknik Informatika, Fakultas Teknik, Universitas Negeri Malang.

## Session Objective

To help students understand how to convert a raw scenario into a clear application development idea before starting system design or coding.

## Key Message

Students should not begin with:

> "I want to build an app."

They should begin with:

> "What real problem am I solving, who faces it, and what process should the application improve?"

## Session Flow

| Time | Segment | Focus |
|---:|---|---|
| 0–3 min | Opening | Why students should not start from coding |
| 3–8 min | Common mistake | Students often focus on tools and features before understanding the problem |
| 8–15 min | Scenario-to-problem framework | How to extract users, context, pain points, data, and application scope |
| 15–23 min | Generative AI workflow | Using ChatGPT, Claude, Google Stitch, Google Sheets, and GitHub |
| 23–28 min | Mini activity | Students improve a weak scenario |
| 28–30 min | Closing | Key takeaway and next action |

## Teaching Strategy

The session should be practical, direct, and example-driven. Students should leave with a simple framework they can reuse in their own application development projects.

## Expected Student Output

At the end of the session, students should be able to produce a short structured scenario containing:

1. Target user
2. Application context
3. Current process
4. Pain point
5. Impact
6. Required data
7. Proposed application support
8. Simple functional requirements
