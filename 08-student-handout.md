# Student Handout

## From Scenario to Application

Before building an application, you must understand the problem.

Do not begin with:

> I want to build an app.

Begin with:

> What real problem am I solving?

## The Scenario Formula

Use this formula:

> User + Context + Current Process + Problem + Impact + Data + Proposed Application

## Example

Weak idea:

> I want to build a complaint system for students.

Better scenario:

> Students often report facility problems such as broken projectors, Wi-Fi issues, damaged chairs, or air-conditioning problems through informal channels such as WhatsApp. These reports may be missed, duplicated, or delayed because there is no structured tracking process. A complaint management application can help students submit complaints, allow staff to update status, and help the faculty monitor unresolved issues using simple report data.

## Checklist Before Building

Before coding, answer these questions:

1. Who is the user?
2. Where does the problem happen?
3. How is the process done now?
4. What is difficult, slow, risky, or inefficient?
5. What data is involved?
6. What happens if the problem is not solved?
7. How can an application help?
8. What is the simplest version of the application?

## How AI Can Help

You can use ChatGPT or Claude to:

1. Clarify your scenario
2. Identify users
3. Write a problem statement
4. Generate system requirements
5. Suggest database fields
6. Create user stories
7. Suggest UI screens
8. Prepare GitHub documentation

You can use Google Stitch to:

1. Generate UI ideas
2. Create screen mockups
3. Explore layout alternatives

You can use Google Sheets to:

1. Store simple prototype data
2. Design early database fields
3. Test simple workflows

You can use GitHub to:

1. Store your code
2. Document your project
3. Track progress
4. Show your development process

## Final Reminder

A good application is not built from coding skills alone. It is built from a clear understanding of the problem.
